Implement the signal-to-strategy registry (P1-R05). Users need a way to create named strategies that generate UUID webhook tokens. Those tokens authenticate incoming TradingView webhooks and route signals to the correct VirtualPortfolio.

The backend lookup (_lookupStrategy in signals.controller.js) already exists and queries signal_strategies/{token} from Firestore. What was missing: no API to CREATE strategies and no UI to manage them.

Output: REST CRUD API at /api/strategies + Strategies management tab in the frontend.
</objective>
